Project Overview

SECS/GEM Communication for Semiconductor Manufacturing

The POLAREX-200 CMP SECS/GEM project implements the SEMI E5 and E30 standards for semiconductor equipment communication, allowing Chemical Mechanical Polishing (CMP) equipment to communicate effectively with factory host systems.

This implementation provides a robust framework for equipment control, monitoring, data collection, and process management in advanced semiconductor manufacturing facilities.

Key Technologies:

  • SEMI E5 SECS-II Message Protocol
  • SEMI E30 GEM Standard
  • SEMI E37 High-Speed SECS Message Services (HSMS)
  • SEMI E37.1 HSMS Single-Session Mode
  • Comprehensive State Models for Equipment Status Tracking
  • Event-Based Data Collection System

SECS Message Implementation

The POLAREX-200 CMP SECS/GEM interface implements a comprehensive set of SECS-II messages according to the SEMI E5 standard. These messages provide equipment status reporting, control capabilities, alarm management, and process data exchange.

Category Messages Description
Equipment Status S1F1/F2, S1F3/F4, S1F11/F12 Basic equipment status monitoring and online verification
Communications Control S1F13/F14, S1F15/F16, S1F17/F18 Establishing and controlling communications between equipment and host
Equipment Control S2F41/F42, S2F13/F14, S2F15/F16 Remote equipment command sending and equipment constant management
Data Collection S2F33/F34, S2F35/F36, S2F37/F38 Configuring and controlling event-based data collection
Alarm Management S5F1/F2, S5F3/F4, S5F5/F6 Equipment alarm reporting and management
Process Programs S7F23/F24, S7F25/F26, S7F19/F20 Managing and transferring process recipes between host and equipment

State Model Implementation

The POLAREX-200 CMP SECS/GEM interface implements five comprehensive state models that provide complete visibility into equipment status and behavior.

Communications State Model

Tracks the communications status between equipment and host:

  • DISABLED: No communications
  • ENABLED: Communications exist
  • NOT COMMUNICATING: Connection established but no active communications
  • COMMUNICATING: Full communications active

Control State Model

Defines the level of control available to the host:

  • OFFLINE: Equipment operation by local operator only
  • ONLINE/LOCAL: Host can obtain data but not control
  • ONLINE/REMOTE: Host has "front panel" control

Process Job State Model

Tracks the lifecycle of processing jobs:

  • QUEUED: Job waiting for execution
  • ACTIVE: Job being processed
  • EXECUTING: Process execution in progress
  • PROCESS COMPLETE: Processing completed

Wafer Process State Model

Tracks the status of individual wafers:

  • NEEDS PROCESSING: Wafer waiting for processing
  • IN PROCESS: Wafer being processed
  • PROCESSING COMPLETE: Wafer processing completed
